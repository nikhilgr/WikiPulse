  "All the world's curiosity, set to music."
  Internal · Concept brief

  A consumer data app · Concept №14
  What is the world reading on Wikipedia right now?
  
    Wikipedia is the closest thing the internet has to a real-time map of human curiosity. Every minute, billions of small decisions about what to look up surface a portrait of what the world is paying attention to. Wiki Pulse renders that portrait — not as a chart, but as a living publication.
  
  
    Concept · Nikhil · Amplitude
    Status · Polished v1 scope
    Inspiration · Flipboard + Wikipedia Pageviews + a slow newsroom aesthetic
  

  № 01
  
    The idea, in one breath
    Vision · One-liner · Hook
  
  
    
      An interactive map of the world, where every country reveals what its people are reading on Wikipedia today, rendered as a beautiful daily publication. Click any country — and read what's on its mind.— The pitch in 30 seconds
    
    
      Most analytics dashboards optimize for utility. Wiki Pulse optimizes for delight. The mechanic is borrowed from cartography (an interactive world map of pageviews-per-country), but the soul is borrowed from print: serif typography, generous whitespace, a sense of ritual.
      The product answers a question almost no one is currently answering well: what is each country curious about, today? What's surging in Japan that nobody in São Paulo cares about? What's the diaspora reading that the home country isn't? What's the one article every country has in common, and what does that tell us?
      It works because Wikipedia is the substrate of curiosity itself, and that curiosity has a geography. People go to Wikipedia because something happened — and what happened in Lagos rarely matches what happened in Lisbon.
    
  

  № 02
  
    Why this works, and why now
    Market · Timing · Wedge
  
  
    
      № 01
      Geography is the story
      The same news event lands differently in different places. Wikipedia's per-country pageview data exposes those differences cleanly — and almost no one is showing them this way.
    
    
      № 02
      The "why" is the moat
      Anyone can list trending articles. The defensible product is the moment of recognition — Russell Crowe spikes in Australia but not in Argentina; we surface the news story or the social moment behind it.
    
    
      № 03
      Open, free, perpetual
      Wikimedia's APIs are free, unauthenticated, and don't sunset. The substrate isn't going anywhere. The cost structure is small enough that delight, not monetization, can lead.
    
    
      № 04
      Editorial aesthetics are back
      Flipboard. Substack. Are.na. The Verge's redesign. There's a renewed appetite for the internet to feel like reading something, not scrolling something.
    
    
      № 05
      Shareability is built in
      "Did you see what spiked on Wikipedia yesterday?" is a tweet, a Slack drop, a screenshot. The product makes its own marketing if the visuals are striking.
    
    
      № 06
      The Amplitude angle
      An Amplitude-adjacent showcase: this is what a consumer-grade data product looks like when treated as a publication, not a dashboard. A craft demo for the brand.

  № 04
  
    Five product principles
    North stars · Design constraints
  
  
    
      
        Principle № 01
        Beauty is a feature
        The product earns the right to exist on aesthetics alone. If a screenshot doesn't make someone stop scrolling, we've failed.
      
      
        Principle № 02
        The "why" is sacred
        Showing what spiked is table stakes. Explaining why something spiked — pulling in news context, related articles, anomaly framing — is the actual product.
      
      
        Principle № 03
        Latency is a design choice
        Hourly is real-time enough. We resist the pull toward second-by-second updates because pageviews are noisy at that resolution, and calm signal beats jittery signal.
      
    
    
      
        Principle № 04
        Less is the design
        One map, one grid, one detail view. We don't add features; we deepen the existing ones. The map is navigation; the grid is the answer; the panel is the story.
      
      
        Principle № 05
        It reads like a publication
        The chrome — masthead, edition number, dateline, sections — is borrowed from print. Visitors should feel like they're opening a daily, not a tool.
      
      "The only feature is the rendering. That's the whole point."

  № 06
  
    The data, end to end
    APIs · Latency · Caching
  

  
    
      Endpoint
      URL · Notes
      Latency
    
    
      
        Top by views (global)
        Daily list of top 1,000 articles by pageviews.
      
      https://wikimedia.org/api/rest_v1/metrics/pageviews/top/en.wikipedia/all-access/{yyyy}/{mm}/{dd}Returns the previous day's top articles by midnight UTC the following day. Filter out Main_Page, Special:*, and namespace-prefixed entries.
      Daily
    
    
      
        Top per country
        Daily list of top 1,000 articles read from a specific country.
      
      https://wikimedia.org/api/rest_v1/metrics/pageviews/top-per-country/{ISO-3166-1-alpha-2}/all-access/{yyyy}/{mm}/{dd}Returns articles with privacy-rounded counts (views_ceil). Only surfaces articles with >100 views in the period. Powers every country click on the map.
      Daily
    
    
      
        Per-article hourly
        Hourly time series for any single article (used in the detail panel).
      
      https://wikimedia.org/api/rest_v1/metrics/pageviews/per-article/en.wikipedia/all-access/all-agents/{article}/hourly/{startYYYYMMDDHH}/{endYYYYMMDDHH}~1-hour lag. Drives the article-level sparkline and "why is this spiking" engine.
      Hourly
    
    
      
        Country boundaries
        TopoJSON world atlas used to render the map.
      
      https://cdn.jsdelivr.net/npm/world-atlas@2/countries-110m.json~100KB. Uses ISO 3166-1 numeric country IDs; we translate to alpha-2 client-side for the API. Loaded once, cached at the CDN edge.
      Static
    
    
      
        Page Summary
        Thumbnail, description, extract, canonical URL.
      
      https://en.wikipedia.org/api/rest_v1/page/summary/{title}Includes thumbnail and originalimage, displaytitle (with italics for film/book titles), short description, and a 200-character extract.
      Static
    
    
      
        EventStreams
        SSE feed of recent changes across Wikimedia.
      
      https://stream.wikimedia.org/v2/stream/recentchangeOptional. Useful for showing a sub-component live edit ticker — "people are editing this article right now" on the detail panel.
      Real-time
    
  

  
    Latency reality check. Wikipedia's "top" and "top-per-country" endpoints are daily and lag by about 24 hours — the freshest pulse you can get is "yesterday's top reads, by country." Per-article hourly (used in the detail panel) lags by about an hour. The product reads as "live" because the map refreshes hourly against the most recent published day; in practice the freshest data point is roughly a day old. This is fine — frame it editorially as "yesterday's pulse, refreshed hourly."
    Rate limits. Wikimedia is rolling out new published rate limits across spring 2026. The historical anonymous ceiling is ~200 req/sec across the REST API. The country-level endpoint is the highest-volume call (one request per country a user explores) — we mitigate by pre-loading the top thirty countries on page paint, then lazy-loading the rest on click. Always set a descriptive User-Agent with a contact address, cache aggressively at the edge, and never re-poll within the freshness window.
    Country-data caveat. The top-per-country endpoint returns views_ceil — privacy-rounded ceiling values, not raw counts — and only surfaces articles that received more than 100 views in the period from that country. Small countries may return short or empty lists. We treat empty results as a real editorial signal ("Wikipedia doesn't surface pageview data for this country today") rather than as an error.

      A static-first architecture. The page is server-rendered with the most recent cached snapshot, then hydrates and refreshes hourly. No database in v1 — KV is enough for the snapshot and the news-context cache.
    
    
      Caching policy
      
        Global top list → Refreshed at :05 past every hour. Cached at the edge with a 30-minute TTL.
        Top per country → Pre-loaded for the top 30 countries on snapshot generation. Other countries fetched lazily on first click, then cached for the lifetime of the snapshot.
        World atlas → Loaded once per page, ~100KB. Cached at the CDN forever (it doesn't change).
        Page Summary → Cached 24 hours (changes rarely). LRU-evicted by usage so popular articles stay warm.
      
    
  

  № 08
  
    The "why" engine
    Spike detection · Context · Editorial framing
  
  
    
      The defining feature of Wiki Pulse isn't the chart. It's the moment of recognition when a user sees a name spike and the product tells them why.
      The "why" is calculated in three layers, each independent and each adding signal.
    
    
      
        Layer 01 · Numerical
        Anomaly framing
        Compare the last six hours of pageviews to the baseline of the prior 42 hours. A ratio above 2× becomes "Climbing fast." Above 4× becomes "Massive surge." A rank delta above +20 becomes "Vaulted X ranks."
      
      
        Layer 02 · Editorial
        News context
        For each surging article, query a news API (NewsAPI, GDELT, or a curated RSS pool) with the article title scoped to the last 24 hours. Surface the highest-cited headline. "Russell Crowe spikes 5.4×. Top story: 'Crowe makes surprise appearance at Cannes.'"
      
      
        Layer 03 · Wikipedia-native
        Edit & link signal
        Surging articles often see editing activity. EventStreams tells us when a top article was edited in the last hour. Wikidata tells us if the article is connected to a current event (an election, an awards show, a death).

  № 11
  
    Risks & open questions
    What could go wrong
  
  
    
      Bot contamination
      Wikipedia pageviews include bot traffic. The "all-access" filter helps but isn't perfect. Mitigation: use the "user" agent filter for spikes detection (filters out automated requests).
      Medium
    
    
      Rate-limit changes
      Wikimedia is rolling out new rate limits through April–May 2026. Our load is small but we need to monitor; consider Wikimedia Enterprise's free tier as fallback for high-traffic spikes on launch day.
      Medium
    
    
      Daily lag
      The "top" endpoint is yesterday's data, not today's. Some users will notice. Frame it editorially — "Yesterday's pulse, refreshed hourly" — rather than apologizing for it.
      Low
    
    
      Editorial responsibility
      If a controversial figure spikes, we surface them prominently. Build a manual override list, with transparent reasoning, for cases where the algorithm clearly fails (e.g., harassment campaigns gaming pageviews).
      High
    
    
      Country-data gaps
      The top-per-country endpoint only surfaces articles with >100 views in the period, and returns privacy-rounded counts. Small countries may have very short top lists or none at all. Frame empty results editorially ("Wikipedia doesn't surface enough signal for this country today") rather than as a bug.
      Medium
    
    
      Visual fatigue
      A world map that always looks the same becomes noise. Tactic: a small "Most-changed since yesterday" indicator on countries whose top article moved. Plus daily editorial framing in v1.5 so each visit has a fresh narrative even when the map looks similar.
      Low
    
    
      "Why" failure
      Sometimes nothing happened — an article just trends because of seasonal patterns or external linking. The product needs grace for "no notable event" cases. Default copy: "Holding steady in the public consciousness today."
      Medium

  № 14
  
    Open questions for decision
    Asks before greenlight
  
  
    
      Domain · wikipulse.live or thepulse.app?
      The brief reserves wikipulse.live; thepulse.app reads more like a publication and travels better in print contexts. Recommendation: secure both, lead with wikipulse.live for v1.
    
    
      News-context API choice
      NewsAPI is cheapest; GDELT is free but harder to parse; a custom RSS pool is most controllable. Recommendation: GDELT for v1.5 (free, fast, structured), graduate later if needed.
    
    
      Multi-language at launch?
      English-only is faster to ship. But the "world's curiosity" framing only fully delivers across languages. Recommendation: English at v1, surface a "Coming soon" toggle, add Spanish + Japanese in v1.5.
    
    
      Brand: standalone or Amplitude-adjacent?
      Wiki Pulse can be a pure consumer brand, or a "by Amplitude" project. The publication aesthetic argues for standalone. Recommendation: standalone brand, Amplitude credit in the colophon.

  };
  const PRELOAD = ['US','GB','IN','DE','JP','BR','FR','IT','ES','CA','AU','MX','RU','NL','ID','PL','TR','SE','KR','NO'];
  const IS_REAL = (a) => a && !a.startsWith('Special:') && !a.startsWith('Wikipedia:') && !a.startsWith('Portal:') && !a.startsWith('File:') && !a.startsWith('Category:') && !a.startsWith('Help:') && !a.startsWith('Talk:') && a !== 'Main_Page' && a !== '-';
  const SHORT = (n) => n >= 1e6 ? (n/1e6).toFixed(1)+'M' : n >= 1e3 ? (n/1e3).toFixed(1)+'k' : String(n);
  const pad = (n) => String(n).padStart(2,'0');

  function reportDate() {
    const d = new Date(Date.now() - 36 * 3600 * 1000);
    return { y: d.getUTCFullYear(), m: pad(d.getUTCMonth()+1), dd: pad(d.getUTCDate()) };
  }

  async function fetchCountryTop(code) {
    const r = reportDate();
    const url = `https://wikimedia.org/api/rest_v1/metrics/pageviews/top-per-country/${code}/all-access/${r.y}/${r.m}/${r.dd}`;
    try {
      const res = await fetch(url);
      if (!res.ok) return null;
      const data = await res.json();
      const articles = (data.items?.[0]?.articles || [])
        .filter(a => IS_REAL(a.article))
        .filter(a => !a.project || a.project.startsWith('en'))
        .map(a => ({ article: a.article, views: a.views_ceil || a.views || 0, rank: a.rank }))
        .slice(0, 10);
      return articles;
    } catch (e) { return null; }
  }

  const cache = new Map();
  const countryNames = new Map();

  function showCountryList(name, articles) {
    const out = document.getElementById('liveCountryOut');
    document.getElementById('liveCountryName').textContent = name;
    const list = document.getElementById('liveCountryList');
    if (!articles || articles.length === 0) {
      list.innerHTML = `<li style="font-style:italic;color:rgba(246,241,232,0.5);">Wikipedia doesn't surface country-level pageviews for ${name}.</li>`;
    } else {
      list.innerHTML = articles.slice(0, 10).map(a => `<li>${a.article.replace(/_/g, ' ')}</li>`).join('');
    }
    out.classList.add('show');
  }

  async function chunkedFetch(items, fn, size = 4) {
    for (let i = 0; i < items.length; i += size) {
      await Promise.all(items.slice(i, i + size).map(fn));
    }
  }

  async function renderMap() {
    const topoUrl = 'https://cdn.jsdelivr.net/npm/world-atlas@2/countries-110m.json';
    const topo = await fetch(topoUrl).then(r => {
      if (!r.ok) throw new Error('topojson load failed');
      return r.json();
    });
    const countries = topojson.feature(topo, topo.objects.countries).features;
    countries.forEach(c => countryNames.set(String(c.id).padStart(3,'0'), c.properties.name));

    const stage = document.getElementById('liveStage');
    const svg = d3.select('#liveWorld');
    svg.style('display', 'block');
    svg.selectAll('*').remove();
    const width = stage.clientWidth - 56;
    const height = 360;
    svg.attr('viewBox', `0 0 ${width} ${height}`);

    const projection = d3.geoNaturalEarth1().fitSize([width, height - 20], { type: 'Sphere' });
    const path = d3.geoPath(projection);

    svg.append('path').attr('class', 'outline').attr('d', path({ type: 'Sphere' }));
    svg.append('path').attr('class', 'graticule').attr('d', path(d3.geoGraticule10()));

    const tip = document.getElementById('liveTip');

    svg.selectAll('path.country')
      .data(countries)
      .enter()
      .append('path')
      .attr('class', d => {
        const num = String(d.id).padStart(3, '0');
        const a2 = NUM_TO_A2[num];
        return 'country' + (PRELOAD.includes(a2) ? ' has-data' : '');
      })
      .attr('d', path)
      .attr('data-num', d => String(d.id).padStart(3, '0'))
      .on('mousemove', function(ev, d) {
        const num = String(d.id).padStart(3, '0');
        const a2 = NUM_TO_A2[num];
        const c = a2 ? cache.get(a2) : null;
        let top = '';
        if (c && c.length > 0) {
          top = `<span class="t">${d.properties.name}</span><span class="v">Top: ${c.slice(0,2).map(a => a.article.replace(/_/g,' ')).join(' · ')}</span>`;
        } else {
          top = `<span class="t">${d.properties.name}</span><span class="v">Click to load top reads</span>`;
        }
        tip.innerHTML = top;
        const rect = stage.getBoundingClientRect();
        let left = ev.clientX - rect.left + 14;
        if (left + 280 > rect.width) left = ev.clientX - rect.left - 296;
        tip.style.left = left + 'px';
        tip.style.top = (ev.clientY - rect.top - 24) + 'px';
        tip.style.opacity = 1;
      })
      .on('mouseleave', () => { tip.style.opacity = 0; })
      .on('click', async function(ev, d) {
        const num = String(d.id).padStart(3, '0');
        const a2 = NUM_TO_A2[num];
        d3.selectAll('.live-stage svg.world path.country').classed('selected', false);
        d3.select(this).classed('selected', true);
        if (!a2) {
          showCountryList(d.properties.name, null);
          return;
        }
        let articles = cache.get(a2);
        if (!articles) {
          articles = await fetchCountryTop(a2);
          if (articles) cache.set(a2, articles);
        }
        showCountryList(d.properties.name, articles);
      });

    // Pre-load top countries for hover previews
    document.getElementById('liveStatusText').textContent = 'Pre-loading top countries…';
    await chunkedFetch(PRELOAD, async (code) => {
      if (cache.has(code)) return;
      const articles = await fetchCountryTop(code);
      if (articles) cache.set(code, articles);
    }, 4);

    document.getElementById('liveStatusText').textContent = 'Streaming · Hover any country';
    document.getElementById('liveSubtitle').textContent = `${countries.length} countries · Click any one to load its top reads`;
  }

  async function boot() {
    const stage = document.getElementById('liveStage');
    const loading = document.getElementById('liveLoading');
    try {
      await renderMap();
      loading.style.display = 'none';
    } catch (e) {
      console.warn('Wiki Pulse live preview fallback:', e);
      stage.classList.add('errored');
      document.getElementById('liveSubtitle').textContent = 'Network unreachable — showing illustrated preview';
      document.getElementById('liveStatusText').textContent = 'Offline mode';
      loading.style.display = 'none';
    }
  }

  if (document.readyState === 'loading') {
    document.addEventListener('DOMContentLoaded', () => setTimeout(boot, 150));
  } else {
    setTimeout(boot, 150);
  }
})();
